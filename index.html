Project Info

Description: The goal of this project is to create a tool to visualize a beginner's algorithm used in the solving of a Rubik’s Cube. We want to accomplish this by having a 3D model of the Rubik’s Cube along with other relevant visual aids. This will allow the user to both interact with the cube and select different steps of the algorithm used throughout the process of solving the cube. This project approaches computational geometry through the idea that a rubik’s cube is both a 3D structure and can be represented in various ways including graph structures. We hope the user is able to understand the principles behind solving the Rubik’s Cube puzzle.

Background: The key problem we are trying to solve, is that of being able to visualize specific algorithms. Many websites will provide a general idea on how to solve a Rubik's Cube but, not work through one with you. Our goal was to allow a user to enter a Rubik's Cube of their own or watch as one gets solved step by step. We were only able to implement the beginner's algorithm currently, so a user would need to use a connected webpage which can provide a stronger understanding of what the algorithm is thinking.

Our approach was a split method of keeping a 3D visualization and backend model, while deciding to have them seperated, that being chaning the backend model does not influence the front end. This is done so that animation can be handled separately, and we bridge the gap by updating the front-end when appropriate. This way we can easily access the backend model in order to implement the algorithms we would like. Otherwise using just the 3D model especially with timed animation would lead to several issues.

Psuedo Code: This application shows the steps for the beginners algroithm. Which consits of solving the white cross, by completing one edge at a time. Then the white corners, also solving each corner piece being careful not to affect neighboring pieces. Next, would be the middle edges where each edge is solved independently by using a repeating algorithm. Subsequently, the yellow cross is solved and then appropriately sorted together. Finally, corners are placed in their appropriate location and made to face upwards one by one. All of these steps get highlighted and animated in our webpage.

Interesting Choices: The biggest choice we made regarding our implementation is that we were able to add in information along side of each step and allow the user to combine steps together when appropriate.
